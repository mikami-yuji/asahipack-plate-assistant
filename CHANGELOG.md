# CHANGELOG

## [1.2.1] - 2026-06-11

### Fixed
- フォルダ選択時に日本語を含むパスが文字化け（mojibake）する問題を解決
  - PowerShellの出力を UTF-8 バイト列から Base64 エンコードして送信し、Node.js 側でデコードするよう修正
- フォルダ選択画面がツールの背面に隠れてしまう問題を解決
  - 最前面（TopMost=true）のダミーウィンドウを親に設定してダイアログを起動させるよう修正

## [1.2.0] - 2026-06-11

### Removed
- 得意先向けExcelファイル（「落版確認書.xlsx」）から「継続の場合の理由」列（旧N列）を削除

### Changed
- メール下書き文面から、継続理由の入力を促す案内文（「（「継続」をご希望される場合は、差し支えなければその理由もご記入いただけますと幸いです）」）を削除

## [1.1.0] - 2026-06-11

### Added
- `pkg` を用いたWindows用スタンドアロン実行可能ファイル (`asahipack-plate-assistant.exe`) のビルド構成を追加
  - Node.jsやその他ランタイムが未インストールのPCでも、ダブルクリックするだけでローカルサーバーが起動しブラウザから利用可能
  - 静的エクスポートファイル (`out/`) をすべてEXE内部に内包
- `japanese-holidays` パッケージの TypeScript 型定義ファイルを追加 (`src/types/japanese-holidays.d.ts`)

### Changed
- `package.json` の `japanese-holidays` バージョン指定エラーを修正 (`^1.1.2` から存在する最新バージョン `^1.0.10` へ変更)
- `README.md` にスタンドアロンEXEの起動方法およびビルド手順を追加

## [1.0.0] - 2026-06-11

### Added
- プロジェクトの初期化 (Next.js App Router, TypeScript, Vanilla CSS)
- `xlsx` および `exceljs` を用いた「落版候補リスト.xlsx」の解析処理を追加
- 顧客用落版確認Excelの生成
  - 指定された表示項目（受注No., Kg, 仕上, 種別, 店名, 銘柄, 色数, 色, 最終使用日, 落版予定日）への絞り込み
  - 仕入先列に基づく「種別」列（シルク印刷/SP/オフセット/カット/アサヒパック/別注）の自動判定と追加
  - 種別が「SP」「シルク印刷」「オフセット」の場合のみ、落版予定日を当月の「最終営業日」（土日祝を除く）に自動変更する機能を実装
  - 元ExcelのB2セルから担当者名（例: 見上、佐藤など）を自動抽出し、Excelの署名（K3セル）や連絡メールの下書き署名に動的反映する機能を追加
  - 回答欄のドロップダウン（廃棄/継続）および継続理由の入力欄の追加
  - 粗利額などの社内情報の除外
- 共有画像フォルダ (`\\asahipack01\画像`) から受注No.に一致する画像の自動検索とコピー機能を追加
- デスクトップ上への得意先別仕分けフォルダの自動作成処理を実装
- 各得意先ごとの連絡メール下書き（件名・本文）の自動作成とクリップボードコピー機能を実装
- Excel生成ロジック (`excelUtils.test.ts`) およびメール生成ロジック (`emailUtils.test.ts`) のユニットテストを追加
- セットアップ手順などを記載した `README.md` の作成
